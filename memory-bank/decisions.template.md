# 架构决策记录 (ADR — Architecture Decision Records)

> 🧊 COLD 知识 — 仅在需要理解"为什么这么设计"或需要推翻旧决策时读取。
>
> **使用方式**: 将本文件复制为 `decisions.md`，然后删除本文件（或保留作为格式参考）。

---

## ADR 模板

```markdown
### ADR-000: [决策标题]

**日期**: YYYY-MM-DD HH:mm（格式见 `progress.template.md` 跨平台时间戳获取说明，禁止猜测）
**状态**: [提议 / 已采纳 / 已废弃]

**背景**
[为什么需要做这个决策？]

**决策**
[我们决定怎么做？]

**备选方案**
- 方案 A: [描述] — 优点/缺点
- 方案 B: [描述] — 优点/缺点

**后果**
[这个决策带来了什么正面/负面影响？]
```
